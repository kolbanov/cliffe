# CS2 Telegram Moderator Bot

Бот для Telegram-беседы:

- ловит политические стоп-слова и обходы вроде `CB0`, `Х0х0л`, `зззеленский`, `путтин`;
- не склеивает слова через пробелы, поэтому `с в о` и `мерс в окне` не считаются `сво`;
- за политические стоп-слова выдаёт предупреждения;
- при достижении лимита выдаёт мут на заданное количество дней;
- ASU-слова не дают предупреждений и мутов — бот просто отвечает `АСУ!!!`;
- если в сообщении есть и политическое стоп-слово, и ASU-слово, приоритет у политического стоп-слова;
- автомодерация политических слов работает для всех участников, включая админов группы;
- ASU-реакция работает на всех, включая админов;
- `/bite` доступна только админам группы и выдаёт предупреждение ответом на сообщение пользователя;
- `/hp` доступна всем участникам группы и показывает текущие хп/предупреждения;
- настройки, предупреждения и реплики бота сохраняются в локальных JSON-файлах после перезапуска.

## Технический стек

- Python 3.13.12
- aiogram 3.28.2
- python-dotenv
- локальные JSON-файлы в папке `data/`

## Быстрый запуск

1. Создай бота через `@BotFather` и получи токен.
2. В `@BotFather` отключи privacy mode: `/setprivacy` → выбери бота → `Disable`.
3. Добавь бота в группу.
4. Назначь бота админом с правами:
   - читать сообщения;
   - ограничивать участников.
5. Напиши боту в личку `/start`, чтобы он мог отправлять тебе админ-панель.
6. В файле `.env` замени токен и укажи ID групп через запятую:

```env
TG_BOT_API_KEY=PASTE_TELEGRAM_BOT_TOKEN_HERE
BOT_TIMEZONE=Europe/Moscow
TG_GROUP_CHAT_IDS=-1001111111111,-1002222222222
```

`TG_GROUP_CHAT_IDS` — это явный список групп для панели. Так бот будет показывать тестовую и основную группу, даже если после обновления он ещё не получил из них события.

7. Если не знаешь ID группы, временно напиши в нужной группе `/chatid`.
8. Если после запуска `/panel` всё равно не видит группу, напиши в группе `/connect` один раз. Бот запомнит группу и попробует удалить команду.
9. Установи зависимости:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

10. Запусти бота:

```bash
python main.py
```

## Команды

### `/panel`

Открывает админ-панель в личной переписке с ботом.

Основной сценарий:

1. Добавь бота в группу.
2. Сделай его админом.
3. Напиши боту в личку `/start`.
4. Напиши боту в личку `/panel`.
5. Выбери нужную группу, если бот подключён к нескольким группам.

Команду не нужно писать в группе. Основной способ подключения двух групп — переменная `TG_GROUP_CHAT_IDS` в `.env`. Если бот после запуска всё равно не видит группу в личке, используй одноразовую команду `/connect` в группе. Она только привязывает группу к панели: бот запомнит чат, попробует отправить панель в личку и удалить команду из группы.

Если всё же написать `/panel` в группе, бот отправит панель в личку как fallback, но настройки рассчитаны именно на приватный сценарий.

В панели есть разделы:

- политические стоп-слова;
- ASU-слова;
- предупреждения;
- сообщение в чат от лица бота;
- реплики бота;
- настройки наказаний.


### `/connect`

Одноразовая команда для привязки группы к личной панели, если бот не видит группу в `/panel`.

Используется в группе только админом:

```text
/connect
```

Бот запоминает группу, пытается отправить панель в личку и через несколько секунд удаляет команду и свой ответ, чтобы не шуметь в чате.


### `/hp`

Команда для всех участников группы. Показывает текущее количество хп и предупреждений:

```text
/hp
```

Ответ будет примерно таким:

```text
у тебя 60хп (2/5 предупреждений)
```

Команда работает в группе и не требует админских прав.


### `/chatid`

Вспомогательная команда для получения ID группы. Используется в группе:

```text
/chatid
```

Бот ответит примерно так:

```text
chat_id этого сервера: -1001234567890
```

Эти значения можно вписать в `.env`:

```env
TG_GROUP_CHAT_IDS=-1001234567890,-1009876543210
```

### Реплики бота

В личной панели выбери нужную группу и нажми **🎲 Реплики бота**.

Там можно менять фразы для:

- предупреждений;
- мутов;
- ASU-реакции;
- ошибки мута, если бот не смог ограничить пользователя.

Для каждой категории доступны действия:

- **➕ добавить** — добавить одну или несколько фраз;
- **🧹 заменить весь список** — полностью заменить текущий набор;
- **🗑 удалить #...** — удалить конкретную фразу по номеру;
- **♻️ вернуть дефолт** — вернуть стандартные фразы.

Telegram-форматирование сохраняется. То есть можно выделить текст жирным/курсивом, сделать code или ссылку прямо в клиенте Telegram, отправить эту фразу боту, и он сохранит её в HTML-виде. Лучше использовать встроенное форматирование Telegram, а не писать HTML руками.

Если отправить несколько строк, каждая непустая строка будет отдельной случайной фразой.

### Отправка сообщения в чат от лица бота

В личной панели выбери нужную группу и нажми **📣 Сообщение в чат**.

Сценарий:

1. Бот попросит прислать текст.
2. Ты отправляешь текст в личку боту.
3. Бот показывает превью и кнопки подтверждения.
4. После нажатия **✅ отправить** бот публикует текст в выбранную группу от своего имени.

Перед отправкой есть подтверждение, чтобы случайно не отправить текст в основную группу вместо тестовой.

### `/bite`

Команда только для админа. Используется ответом на сообщение пользователя.

Если написать не ответом, бот скажет:

```text
кого кусать?
```

### `/pol` и `/asu`

Команды только для админов группы. Быстро добавляют слова в словари прямо из чата.

Добавить слово или несколько слов:

```text
/pol хохляндия хохляндец
/asu нигретос
```

Или ответить командой на сообщение — бот возьмёт слова из текста/подписи сообщения:

```text
/pol
/asu
```

После добавления matcher пересобирается сразу, перезапуск не нужен.

## Логика предупреждений

Дефолт:

- 5 предупреждений до мута;
- мут на 1 день.

HP считается автоматически:

```text
0 предупреждений из 5 = 100хп
1 предупреждение из 5 = 80хп
2 предупреждения из 5 = 60хп
3 предупреждения из 5 = 40хп
4 предупреждения из 5 = 20хп
5 предупреждений из 5 = 0хп и мут
```

## Файлы данных

### `data/stop_words_politics.json`

Словарь политических стоп-слов.

### `data/asu_words.json`

Словарь ASU-слов.

### `data/letter_mapping.json`

Маппинг букв и визуальных замен.

### `data/settings.json`

Настройки лимитов и длительности мута.

### `data/warnings.json`

Предупреждения пользователей.

### `data/known_chats.json`

Группы, которые бот уже видел. Если группы прописаны в `.env` через `TG_GROUP_CHAT_IDS`, бот также добавит их сюда при запуске.

### `data/phrases.json`

Случайные фразы бота для предупреждений, мутов, ASU-реакции и ошибки мута.

## Важная логика фильтра

Между буквами разрешены символы вроде:

```text
-, ., _, *, @
```

Но не разрешены обычные пробелы.

Сработает:

```text
CB0
C-B-0
с.в.о
Х0х0л
зззеленский
путтин
```

Не сработает:

```text
с в о
мерс в окне
мерс-в-окне
украсть
украл
украинец
украина
оркестр
```


## Диагностика .env и групп

Если `/panel` не видит группы, напиши боту в личку:

```txt
/config
```

Команда покажет, загрузился ли `.env`, что лежит в `TG_GROUP_CHAT_IDS`, какие ID распарсились и видит ли бот тебя админом в этих группах.

`.env` теперь грузится не только из текущей рабочей папки, но и из папки, где лежит `main.py`, поэтому запуск из другой директории не ломает чтение переменных.
